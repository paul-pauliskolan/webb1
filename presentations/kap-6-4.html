Modul 6 · Kapitel 6.4

Prestanda och enkel felsökning

Mät och läs felmeddelanden innan du börjar gissa.

Mål

Efter kapitlet ska du kunna...

  • hitta tunga och saknade filer i Network
  • förstå vanliga orsaker till långsam laddning
  • använda Lighthouse som stöd
  • felsöka fel sökväg och filnamn

Varför?

En snabb sida hjälper fler

  • Mobiltelefoner kan ha långsammare uppkoppling.
  • Stora filer tar tid och använder mer data.
  • Saknade filer gör att innehåll eller funktioner försvinner.

Statuskoder

200 och 404 betyder olika saker

200

Filen hittades och kunde hämtas.

404

Filen hittades inte på den angivna sökvägen.

När en fil ger 404

Kontrollera namnet exakt

<img src="images/Bild.webp" alt="...">

Men filen heter:

images/bild.webp

Bild.webp och bild.webp kan räknas som olika filnamn på webbservern.

Stora filer

Sortera kolumnen Size

  1. Öppna Network.
  2. Ladda om sidan.
  3. Klicka på Size.
  4. Undersök de största filerna först.

Bilder, video, ljud och externa inbäddningar är ofta stora.

Testa utan cache

Hämta filerna på nytt

  1. Öppna DevTools och Network.
  2. Markera Disable cache.
  3. Låt DevTools vara öppet.
  4. Ladda om sidan.

På Mac kan du även göra en hård omladdning med ⌘ + ⇧ + R.

Lighthouse kan ge tips inom:

Performance
Accessibility
Best Practices

Välj mobil eller desktop och kör analysen.

Inte ett facit

Använd Lighthouse som stöd

  • Resultatet kan variera mellan körningar.
  • Dator, nätverk, cache och externa tjänster påverkar.
  • Välj ett konkret problem.
  • Ändra, testa och kör analysen igen.

Console

Leta efter röda fel

  1. Öppna Console.
  2. Ladda om sidan.
  3. Läs felmeddelandet.
  4. Kontrollera filnamn och radnummer.

Vanliga orsaker är saknad JavaScript-fil, saknat element eller kod som körs för tidigt.

Elements

Undersök HTML och CSS

  1. Högerklicka på elementet och välj Inspektera.
  2. Läs HTML-strukturen till vänster.
  3. Läs CSS-reglerna till höger.
  4. Slå av en CSS-regel och se vad som ändras.

Mobilvy

Testa smalt och brett

Öppna mobilvyn på Mac med:

⌘ Command + ⇧ Shift + M
  • Kontrollera text, bilder och navigation.
  • Kontrollera video och formulär.
  • Testa även bred desktopvy igen.

Felsök i ordning

En kontrollerad arbetsgång

  1. Kontrollera Console för fel.
  2. Kontrollera Network för saknade eller stora filer.
  3. Testa smal och bred viewport.
  4. Ändra en sak och testa igen.

Före publicering

Kontrollera sidan

  • Inga röda fel i Console.
  • Inga 404-filer i Network.
  • Rimliga filstorlekar.

  • Lighthouse-problem är granskade.
  • Testat utan cache eller med hård omladdning.

Sammanfattning

Kom ihåg

  • Network visar filer, storlek och status.
  • 404 betyder att filen inte hittades.
  • Console visar fel i JavaScript.
  • Elements hjälper med HTML och CSS.
  • Lighthouse ger tips, inte ett slutgiltigt facit.
  • Ändra en sak i taget och testa igen.

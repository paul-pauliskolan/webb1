Startfiler

Ladda ner och packa upp

  1. Ladda ner ovn2.zip.
  2. Packa upp zip-filen.
  3. Kontrollera att mappen heter ovn2.
  4. Öppna bara mappen ovn2 i VS Code.

Filer

Det här finns i mappen

ovn2/
├── index.html
├── html.webp
└── css.webp

CSS-filen finns inte ännu. Du ska skapa den med det namn som står i HTML-filens befintliga <link>.

Viktig regel

Ändra inte link-elementet

<link rel="stylesheet" href="styles.css">

Skapa därför filen styles.css.

Ändra inte länken så att den passar ett annat filnamn.

HTML-rättning

Rätta bara felen

  • Kör index.html genom HTML-validatorn.
  • Läs ett fel i taget.
  • Rätta felet i VS Code.
  • Validera igen.

Ändra inte texter eller struktur bara för att du själv hade skrivit dem annorlunda.

Kontrollera bilder

Filnamnet måste stämma exakt

<img src="css.webp" alt="CSS-logotyp">
<img src="html.webp" alt="HTML-logotyp">
  • Filändelsen ska matcha filen i mappen.
  • Varje innehållsbärande bild behöver relevant alt-text.

CSS-krav 1

Font och text

body {
  font-family: Verdana, sans-serif;
  color: #000;
}

All text ska använda Verdana med sans-serif som reserv.

CSS-krav 2

Två ljusa bakgrunder

body { background-color: #f3f0dc; }
article,
footer { background-color: #d7edf1; }
  • body ska vara ljus men inte vit.
  • article och footer ska ha en annan ljus färg.

CSS-krav 3

Länkarnas två lägen

a {
  color: #075985;
  text-decoration: none;
}
a:hover {
  text-decoration: underline;
}

Länken ska vara mörkare än bakgrunden och få understreck vid hover.

CSS-krav 4

Två spalter med Grid

section {
  display: grid;
  grid-template-columns: 1fr 1fr;
  gap: 1rem;
}

De två article-elementen blir Grid-items.

Alternativ

Två spalter med Flexbox

section {
  display: flex;
  gap: 1rem;
}
article { flex: 1; }

Välj Grid eller Flexbox. Du behöver inte använda båda för spaltkravet.

Validera HTML

W3C Markup Validation Service

  1. Öppna HTML-validatorn.
  2. Ladda upp eller klistra in index.html.
  3. Rätta fel – inte bara varningar du inte förstår.
  4. Kör kontrollen igen tills HTML-koden klarar valideringen.

Validera CSS

W3C CSS Validation Service

  1. Öppna CSS-validatorn.
  2. Ladda upp eller klistra in styles.css.
  3. Rätta syntaxfel.
  4. Kör kontrollen igen tills CSS-koden klarar valideringen.

Publicera kod

GitHub-repository

  • Lägg filerna i ett repository.
  • Kontrollera att index.html, styles.css och bilderna finns där.
  • Öppna filerna på GitHub och kontrollera senaste versionen.

Publicera sida

GitHub Pages

  1. Öppna repositoryts Settings.
  2. Välj Pages.
  3. Publicera från rätt branch och mapp.
  4. Öppna den publicerade adressen och testa sidan.

Slutkontroll

Kontrollera kraven

  • Verdana och svart text
  • Två olika ljusa bakgrunder
  • Mörka länkar utan understreck – understreck vid hover
  • Två spalter med Grid eller Flexbox
  • HTML och CSS validerade

Inlämningschecklista

Allt ska vara klart

  • Mappen heter ovn2.
  • HTML-felen är rättade utan onödiga ändringar.
  • styles.css är skapad.
  • Bilder och länkar fungerar.
  • GitHub- och GitHub Pages-länkarna är testade.

Arbetsordning

Gör en sak i taget

  1. Packa upp och öppna mappen.
  2. Rätta och validera HTML.
  3. Skapa CSS och uppfyll kraven.
  4. Validera CSS.
  5. Publicera och testa båda länkarna.
